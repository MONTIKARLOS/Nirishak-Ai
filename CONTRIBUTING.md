# Contributing to Nirikshak AI

Thank you for your interest in contributing! 🎉

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/Nirikshak-AI.git`
3. Create a feature branch: `git checkout -b feat/your-feature-name`
4. Make your changes
5. Commit with conventional commits: `git commit -m "feat: add new feature"`
6. Push and open a Pull Request

## Commit Message Convention

We follow [Conventional Commits](https://www.conventionalcommits.org/):

| Prefix | Description |
|--------|-------------|
| `feat:` | New feature |
| `fix:` | Bug fix |
| `docs:` | Documentation |
| `chore:` | Maintenance |
| `test:` | Tests |
| `refactor:` | Code refactor |

## Code Style

- **Frontend**: Follow React best practices, use functional components + hooks
- **Backend**: Follow Spring Boot conventions, use constructor injection
- **Python AI**: Follow PEP8, add type hints

## Branch Naming

- `feat/feature-name` — new features
- `fix/bug-name` — bug fixes
- `docs/doc-name` — documentation updates

## Pull Request Checklist

- [ ] Code follows style guidelines
- [ ] Tests added/updated
- [ ] Documentation updated
- [ ] No sensitive data committed

## Reporting Issues

Use [GitHub Issues](https://github.com/Suraj-kummar/Nirikshak-AI/issues) with:
- Clear title and description
- Steps to reproduce
- Expected vs actual behaviour
- Environment details

## Contact

For questions, reach out via GitHub Issues or Discussions.
